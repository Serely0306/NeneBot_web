# Sakura bot

本部分为Sakura bot的帮助文档，基于[Sakura Bot 帮助中心](https://sakura-bot.cn/#/rhythm-games/project-sekai)，此处仅作整理

用户数据部分可以加上tw/cn前缀来触发国/台服对应功能

部分国/台服功能有独立特性，具体会在功能内说明

没有参数的功能就是可以被at的，使用时请注意

## 账号与绑定

| 指令 | 功能 |
|------|------|
| `绑定+ID` | 绑定游戏账号，ID 可在游戏内个人信息页查看 |
| `切绑定` | 切换已绑定的多个 ID 中的默认 ID |
| `解绑` | 解绑指定 ID |
| `id` | 返回当前 QQ 号绑定的 ID |
| `注册` | 创建新的日服账号并自动完成基础引导流程，仅支持日服，同一时间仅执行一个注册任务 |
| `查时间` | 查询当前绑定 ID 的注册时间 |

绑定后默认为"不给看"模式，无法被 at。已支持多 ID 绑定。

### 给看 / 不给看

开放或关闭其他人 at 查询你的权限，全服通用。

## 个人信息与进度

以下功能除 `pjskprofile`、`视奸`、`逮捕` 外，均需上传数据后才可使用。日服玩家可直接将引继信息复制给 Bot 实现上传（需重新引继一次）。

[点我进入上传数据页面（日/台服）](https://upload.Sakura-bot.cn)

| 指令 | 功能 |
|------|------|
| `pjskprofile` | 获取个人信息图片（类似旧版 profile） |
| `视奸` | 返回当前队伍综合力及其构成 |
| `逮捕` | 查询 EXPERT / MASTER FC 及 AP 进度。已上传数据用户可查高难度 FC/AP 及排位数据 |
| `pjskdetail` | 获取个人详细信息：金币、水晶数量、家具练度、每日挑战进度等。如介意可私聊查询 |
| `pjskcard` | 获取个人卡牌图鉴。可通过 `pjskcard+角色花名` 查看单角色图鉴（支持全服） |
| `pjskevent` | 获取个人 10 次最高排名的活动记录（当期需拿到排名奖励才会记录） |
| `rk` | 获取玩家排位信息 |
| `b39` | 获取 Rating 分数最高的 39 首歌曲和总分（仅供参考） |
| `pjsk b30` | 获取 Rating 分数最高的 30 首歌曲和平均分数（仅供参考），与 b39 分数算法不同 |
| `b30删歌` | 将指定歌曲从 b30 / b39 统计中排除，支持曲名或歌曲 ID |
| `b30恢复歌` | 恢复被排除的歌曲 |
| `b30删歌列表` | 查看当前排除列表 |
| `pjsk进度` | 查询 MASTER 歌曲完成情况（AP / FC / Clear / All），可用 `pjsk进度ex` / `pjsk进度apd` 查 EXPERT / APPEND |

## MySekai 查询

需上传 MySekai 数据后使用。当前仅日服完整支持，TW/CN 预留了指令入口但上传接口和完整业务逻辑尚未实现。

| 指令 | 功能 |
|------|------|
| `msg` / `msgate` | 查询 MySekai Gate 升级材料与进度，可带 gateId 指定目标门。需已绑定并存在对应区服 suite 数据 |
| `msf` / `家具列表` / `家具详情` | 不带参数返回家具总览，输入数字查看家具详情，输入关键字可搜索。已上传数据时会额外展示当前库存和材料缺口 |
| `msr` / `msmap` / `msa` | 查询 MySekai 资源与地图总览，默认隐藏已采收资源。参数带 `all` 可显示全部资源，图片过长时会拆分多张发送 |
| `msp` / `mysekai照片` | 不带参数返回照片列表与预览，输入序号查看指定照片。支持负数倒序（如 `msp -1` 查看最新一张） |
| `msm` / `mss` / `mssong` | 查询 MySekai 音乐记录与收藏情况，参数带 id 时额外显示记录 ID |

## 组卡

| 指令 | 功能 | 示例 |
|------|------|------|
| `组卡` | 返回指定箱活推荐最佳卡组 Top7，按活动点数排序。第 1 参数为队名，第 2 参数为属性（颜色），两个参数之间必须有且只有一个空格。歌曲锁定 EXPERT 独りんぼエンヴィー | `组卡 ln 橙` |
| `活动组卡` | 返回当前活动推荐最佳卡组 Top7。第 1 参数为歌曲名称，第 2 参数为歌曲难度。不输难度默认 EXPERT，不输歌曲默认 独りんぼエンヴィー。WL 活动第 1 参数识别为对应角色 | `活动组卡 龙 hard` · `活动组卡 角色=ick 10th master` |
| `挑战组卡` | 返回每日挑战推荐最佳卡组 Top7，按分数排序。参数依次为角色名（花名 OK）、歌曲名称、歌曲难度。不输难度默认 EXPERT，不输歌曲默认 独りんぼエンヴィー | `挑战组卡 ick 10th master` |
| `最强组卡` | 返回同色同队中分数最高的卡组 Top7，无需参数。歌曲锁定 EXPERT 独りんぼエンヴィー | `最强组卡` |

队名：`ln` `mmj` `vbs` `ws` `25` `vs`
属性：`绿` `粉` `橙` `蓝` `紫`

## 活动与档线

| 指令 | 功能 |
|------|------|
| `活动` | 查看当前活动信息。加编号可查指定活动，如 `活动86` |
| `findevent` | 查询满足指定条件的所有活动。Bot 内使用 `findevent help` 查看帮助 |
| `sk` | 查询活动排名或指定档线。WL 活动时可加角色花名查看单榜排名 |
| `sk预测` | 获取预测线信息（仅日服单榜，数据来源于 33） |
| `分数线` / `sk线` | 获取当前档线分数，最大 5 分钟延迟。WL 活动可 `+角色名` 查看单榜分数线 |
| `ss` | 获取当前档线时速（20 分钟速度 ×3） |
| `wlss` | 获取当前角色单榜档线时速（日服限定） |
| `查房` | 获取本人或目标档线最近 1 小时的活动情况。无法确认烤森刷周回，请自行判断 |
| `5v5胜率` | 5v5 活动返回双方队伍胜率，每 1 小时更新一次 |
| `5v5分数` | 5v5 活动每次结算时获取双方分数 |
| `tf启动` | 5v5 活动中送 3 次无偿火给绑定 ID 用户 |

## 歌曲与角色查询

| 指令 | 功能 |
|------|------|
| `pinfo` | 查询歌曲信息，可使用名称或花名。可通过 `pset+空格+新昵称+空格+to+空格+旧昵称` 设置花名，`pdel+昵称` 删除 |
| `charinfo` | 查询角色所有花名。可通过 `charset` / `chardel` 修改，格式同上 |
| `难度排行` | 返回指定条件下所有歌曲的难度定数排行。格式：`难度排行+level+空格+难度`。可加 `fc` / `ap` 前缀查 FC/AP 定数排行（仅紫谱及早期红谱，2023-2-20 后定数按 AP 难度计算） |
| `谱面预览` | 歌曲名 + `谱面预览`，默认 MASTER 难度。可用 `ex谱面预览` / `apd谱面预览` 查看 EXPERT / APPEND |
| `技能预览` | 歌曲名 + `技能预览`，默认 MASTER 难度。可用 `ex技能预览` / `apd技能预览` 查看 EXPERT / APPEND。返回谱面技能分布及分数占比 |
| `查bpm` | 查询指定 BPM 或变 BPM 歌曲中包含该 BPM 的歌曲 |
| `查物量` | 查询所有歌曲难度中谱面物量等于指定值的歌曲及难度 |

## 卡牌查询

| 指令 | 功能 |
|------|------|
| `findcard` / `查卡` | 返回满足条件的所有卡牌。最多 3 个参数（队名/角色名 → 稀有度/属性），稀有度含 `4` `3` `2` `1` `birthday` `限定`，属性含 `绿` `粉` `紫` `黄` `橙` `蓝` |
| `card` | `card+卡编号` 返回指定卡面图片 |
| `cardinfo` | `cardinfo+卡编号` 返回卡面基础信息（属性、技能、当期活动等） |
| `看` | `看+角色名` 返回该角色随机卡图 |

## 娱乐功能

| 指令 | 功能 |
|------|------|
| `pjsk听歌识曲` | 随机选择音源截取 4-7 秒发送，40 秒内直接输入歌曲名/花名回答。`结束猜曲` 中断 |
| `pjsk猜卡面` | 随机截取 3/4 星卡面 250×250 像素，40 秒内输入角色名/花名回答。`结束猜卡面` 中断 |
| `pjsk抽卡` | 默认 10 连。`pjskXX连` 文字返回结果，`pjsk反抽卡+当期卡池编号` 反转 2/4 星概率。卡池编号可通过 cardinfo 查询 |
| `pjsk表情制作` | 自定义 PJSk 小表情上的文字。`pjsk -h` 查看详细使用说明 |
| `随个` | 随机指定 level 歌曲 info。加组合名可从该组合演唱/翻唱曲中随机 |
| `葱什么` | 返回随机歌曲 info |

## Live 订阅

群主或管理员可为当前群开启或关闭结活 Live 通知推送。兼容指令：`开启live订阅` `开启live推送` `开启live通知` / `关闭live订阅` 等。

每天 10 点还会检测是否有更新，如有更新推送到订阅群聊。
